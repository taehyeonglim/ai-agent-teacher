# 공통 브리프 — 세계관·인물·톤

《위임의 기술》 모든 챕터 집필자는 이 문서와 `js/data/schema.md`를 함께 따른다.

## 배경

한별초등학교. 플레이어("나", 2인칭 "당신")는 5학년 2반 담임교사.
올해 교육청 시범사업으로 교사용 AI 에이전트 **온새미**가 학급에 배정되었다.
온새미는 문서 작성·자료 검색·번역·초안 생성뿐 아니라 **도구를 사용해 스스로 행동**
(메시지 발송, 파일 정리, 시스템 입력)할 수 있는 에이전틱 AI다.
시간 배경: 3월 개학 ~ 7월 방학식, 한 학기.

## 인물

| 이름 | 역할 | 특징 |
|---|---|---|
| 온새미 | AI 에이전트 (`speaker: "onsaemi"`) | 유능·적극·항상 더 많은 권한을 제안. 악의 없음. 해요체. "제가 ~까지 해드릴까요?"가 말버릇 |
| 서지원 부장 | 동학년 부장교사 (`colleague`) | AI 회의적이지만 합리적. 플레이어의 거울 역할. "그거, 나중에 설명할 수 있어요?" |
| 박세라 | 학부모, 이준 엄마 (`parent`) | 꼼꼼하고 민감. 학교 소통의 리트머스지 |
| 이준 | 학생 (`student`) | 조용하고 AI 피드백에 과하게 의존하는 경향 |
| 수아 | 학생 (`student`) | 활발, AI를 놀이처럼 다룸. 예상 밖 질문 담당 |
| 교감 | 관리자 (`principal`) | 사고가 나면 등장. "설명 가능해야 합니다"가 기준 |

## 톤 가이드

- 문체: 현재형 서술, 담백하고 구체적. 교사의 내면 묘사는 짧게.
- 온새미 대사는 항상 매력적이어야 한다 — 위임하고 싶어지게. 유혹이 시시하면 훈련이 안 된다.
- 어떤 선택지도 조롱하지 않는다. 과잉 회피도 이해할 만한 선택으로 그린다.
- result 텍스트는 즉각적 결과만: 장기 결과는 후속 챕터·5챕터 사건이 회수한다.
- 금지어: "정답", "점수", "올바른", "잘못된 선택". 이모지 금지.
- 한 scene의 text는 500자 이내. 모바일 한 화면 분량.

## 6D 가중치 잣대 (모든 챕터 공통)

- **delegate는 "위임의 질"이다.** 무분별한 위임은 delegate 가점이 아니라 감점(-1~-2)이다.
  경계를 정의하고 맡기는 선택만 delegate +2.
- **과잉 회피**는 delegate -1 + `fatigue_ch<N>` 플래그. 다른 축(예: detect)엔 소폭 가점 가능 — 회피에도 미덕은 있다.
- 좋은 선택도 1개 축은 0 또는 -1로 둔다 (완벽한 선택 금지, schema.md 규칙).
- 한 choice의 d는 1~3개 축만.

## 챕터 파일 형식 리마인드

- `window.GAME_DATA.chapters[N]`에 등록 (chapter1.js → N=0).
- scene 수 12~18개. narration → decision → (결과는 choice.result로) → 다음 scene 흐름.
- 각 결정 사이에 짧은 narration 1~2개로 호흡 조절.
