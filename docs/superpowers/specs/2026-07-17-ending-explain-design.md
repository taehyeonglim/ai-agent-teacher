# 결과창 설명 보강 설계 (2026-07-17)

## 배경

v1 배포 후 플레이어 피드백: "마지막 결과창에 설명이 좀 더 있으면 좋겠어요. 각 항목이
뜻하는 바와 점수가 무슨 뜻인지." 현재 결과창은 축 이름과 숫자만 보여 주고, 6D 정의는
온보딩 화면에만 나와 방학식 시점에는 잊힌 상태다. 점수의 기준(무엇 대비 100인지)과
처방 2건의 선정 이유도 화면에 없다.

## 결정 사항 (사용자 확정)

1. **점수 해석 깊이** — 기준 문구 + 구간 해석. 칭호 판정 로직은 공개하지 않는다
   (스포일러 방지, `???` 도감 수집 동기 유지).
2. **정의 표시 방식** — 점수 행마다 정의 한 줄을 항상 표시 (토글·툴팁 아님).
3. **적용 범위** — 본인 결과창(`showEnding`) + 공유 결과 화면(`showSharedResult`)에
   동일 적용. PNG 카드(`saveResultImage`)에는 기준 문구 한 줄만.

## 설계

### 1. 구간 판정 — js/scoring.js

- `BANDS = { high: 60, mid: 35 }` 상수 추가. 에필로그 분기(평균 60/35)와 축별 구간
  해석이 같은 상수를 쓴다.
- `bandOf(value)` 순수 함수 추가: `value >= 60 → 'high'`, `>= 35 → 'mid'`,
  미만 → `'low'`. `Object.freeze` 공개 객체에 `BANDS`, `bandOf` 포함.
- `bootstrap.js`의 에필로그 선택식(`avg >= 60 ? … : avg >= 35 ? …`)을
  `Scoring.BANDS` 참조로 교체한다.

### 2. 점수 리스트 공통화 — js/bootstrap.js

- `scoreListHTML(normalized)` 헬퍼를 추출해 `showEnding`·`showSharedResult` 양쪽에서
  사용한다 (기존 중복 제거).
- 리스트 위 캡션: "각 점수는 모범적으로 한 학기를 보냈을 때를 100으로 환산한 값입니다."
- 각 행: 축 라벨 + 점수 + 구간 배지 + 정의 한 줄.
  - 배지 문구: `high` → "● 안정", `mid` → "◐ 성장 중", `low` → "○ 돌아볼 지점"
    (복기 화면의 "되돌아볼 지점" 표현과 맞춤).
  - 정의 문구는 온보딩과 공용인 `D6_DEFS` 배열을 그대로 재사용한다.
- 기존 `radar-scores`의 `aria-label`은 캡션이 본문에 생기므로 중복되지 않게 정리한다.

### 3. 처방 선정 이유 — js/bootstrap.js

- 처방 목록(`rx-list`) 위에 소제목을 추가: "다음 학기를 위한 처방 — 여섯 축 가운데
  상대적으로 아쉬웠던 두 축입니다."
- 모든 축이 높아도 하위 2축이 나오는 구조에 대한 설명을 겸한다.
- 본인 결과창과 공유 화면 모두 적용 (공유 화면은 30일 실천 없이 summary만 보여 주는
  기존 차이 유지).

### 4. PNG 결과 카드 — js/bootstrap.js `saveResultImage`

- 막대 목록 아래에 기준 문구 한 줄 추가: "각 축은 모범적인 한 학기를 100으로 환산".
- 축별 정의 6줄은 넣지 않는다. 캔버스 크기(800×1060) 유지, 기존 하단 크레딧과
  겹치지 않게 y 좌표만 조정.

### 5. 스타일 — css/game.css

- `.score-caption`(캡션), `.axis-def`(정의 줄), `.band`(배지, high/mid/low 변형),
  `.rx-head`(처방 소제목) 스타일 추가. 기존 종이·올리브 팔레트 변수 재사용.
