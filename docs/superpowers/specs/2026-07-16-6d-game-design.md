# 《위임의 기술》 — 6D 교사 AI 리터러시 훈련 게임 설계서

- 날짜: 2026-07-16
- 기획: Claude Fable 5 (총괄) / 구현: Codex CLI `gpt-5.6-terra` 병렬 외주
- 원천 자료: 「교사를 위한 AI 리터러시」 특강 덱 (claude.ai/design 프로젝트 `fc24068f`, index.html)

## 1. 목적과 대상

AI 시대 교사의 핵심 역량인 **6D 모델**(Define·Design·Delegate·Detect·Decide·Disclose)을
"무엇을 위임하고, 어디서 멈추게 하고, 누가 책임질 것인가"라는 판단 훈련으로 체득시키는
**범용 공개 웹 교구**. 특강과 무관하게 누구나 링크만으로 플레이 가능해야 하며,
온보딩·개념 설명을 게임 안에 포함한다.

### 6D 모델 (원천 정의)

| D | 이름 | 정의 |
|---|------|------|
| 1 | Define 정의 | 교육목표와 인간이 책임질 영역을 정의한다 |
| 2 | Design 설계 | 인간과 AI의 전체 업무 흐름을 설계한다 |
| 3 | Delegate 위임 | AI에 맡길 업무·데이터·도구·권한을 정한다 |
| 4 | Detect 탐지 | 오류·편향·개인정보·저작권·과잉의존을 탐지한다 |
| 5 | Decide 결정 | 승인·수정·중단·재검토를 인간이 결정한다 |
| 6 | Disclose 공개 | AI 사용·출처·과정·책임자를 투명하게 밝힌다 |

출처 표기 의무: UNESCO (2024) · EU AI Act 제4조 · OECD 종합 재구성 실천 모델
(공인 프레임워크 아님) — 게임 크레딧/결과 화면에 명시.

## 2. 컨셉

플레이어는 AI 에이전트 **"온새미"**를 배정받은 초등 5학년 담임교사.
3월 개학부터 여름방학까지 한 학기 동안 수업준비·피드백·학부모소통·평가·위기대응
업무를 처리하며 매 결정마다 위임의 범위와 승인 지점을 선택한다.
점수는 게임 중 숨기고, 선택의 결과는 서사(학부모 민원, 학생 성장, 동료 반응)로 돌아온다.
종료 시 6D 레이더 프로파일 + 맞춤 처방을 제공한다.

형식: **시뮬레이션 내러티브** (분기형 의사결정, Reigns/텍스트 어드벤처 계열).
1회 플레이 약 20분.

## 3. 게임 구조 — 5챕터 + 에필로그

| 챕터 | 시기 | 업무 영역 | 핵심 훈련 D | 원천 슬라이드 |
|---|---|---|---|---|
| 1. 새 학기 | 3월 | 수업 준비 | Define, Delegate | 사례 1 수업 준비 |
| 2. 첫 평가 | 4월 | 개별 피드백 | Design, Detect | 사례 2 개별 피드백 |
| 3. 상담 주간 | 5월 | 학부모 소통 | Delegate, Decide | 사례 3 + 6D 캔버스 라이브 설계 |
| 4. 수행평가 | 6월 | 평가·공개 | Decide, Disclose | 취약한 평가 · AI 사용 공개서 |
| 5. 사건 | 7월 | 위기 대응 | 6D 종합 | 레드팀 6질문 |
| 에필로그 | 방학식 | 결과 정산 | — | 30일 실천 카드 |

- 챕터당 3~4개 결정, 총 17~20개 결정.
- **챕터 5 "사건"**: 앞선 챕터의 선택 플래그가 발화하는 챕터.
  레드팀 6질문(오정보 확신 사용·데이터 외부 잔존·권한 초과 실행·이의 제기·
  사용 미공개·과정 설명 불가)을 사건 형태로 구현. 플레이어의 누적 설계가
  튼튼하면 사건이 방어되고, 허술하면 결과가 악화된다.
- 선택지 설계 원칙: "더/덜 위임" 단일 축 금지. 매 결정은
  **잘 설계된 위임 / 무분별한 위임 / 과잉 회피** 를 포함한 3~4지선다.
  과잉 회피(AI 전면 거부)에도 비용을 부여(교사 피로 누적 → 후속 챕터 파급)하여
  "안 쓰면 정답" 학습을 차단한다.

## 4. 점수 모델

- 선택지마다 6개 D 축에 -2 ~ +2 가중치 (보통 2~3개 축에 영향).
- 게임 중 노출: 서사적 결과 + 은은한 연출 신호(교사 피로도, 교실 분위기)만.
  수치·정답 표시는 절대 없음.
- 플래그 시스템: 선택이 불리언 플래그(예: `roster_leaked`, `no_disclosure`)를 남기고
  챕터 5 사건 분기와 엔딩 서사가 이를 참조한다.
- 엔딩 화면:
  1. 6D 레이더 차트 (SVG 직접 렌더)
  2. 최저 2개 축 맞춤 처방 — 해당 D 개념 요약 + 30일 실천 문장 제안
  3. 플레이 스타일 칭호 (예: 신중한 오케스트레이터 / 고독한 장인 / 브레이크 없는 위임러)
  4. 결과 이미지 저장·공유 버튼 (연수장 비교 활동 유도)

## 5. 기술 아키텍처

순수 정적 웹. 서버·API 키·빌드 도구 없음. GitHub Pages 배포.

```
ai-agent-teacher/
├── index.html          # 셸 (게임 마운트 지점)
├── css/
│   └── game.css        # 특강 덱의 종이 질감·올리브 톤 계승
├── js/
│   ├── engine.js       # 씬 렌더링·분기·상태(플래그)·localStorage 이어하기
│   ├── scoring.js      # 6D 점수 누적·레이더 SVG·칭호/처방 산출
│   └── data/
│       ├── schema.md   # 시나리오 데이터 스키마 정의 (집필 계약서)
│       ├── chapter1.js ~ chapter5.js
│       └── endings.js  # 처방·칭호·에필로그 텍스트
```

- **엔진/콘텐츠 완전 분리**: 시나리오는 선언적 데이터
  `{scene id, 본문, 선택지[{텍스트, 6D 가중치, 플래그, 다음 scene}], 조건 분기}`.
  스키마 하나로 하위모델 병렬 집필 품질을 계약한다.
- 바닐라 JS (프레임워크 없음), 모바일 우선 반응형.
- 상태 저장: `localStorage` (이어하기), 초기화 버튼 제공.

## 6. 개발 역할 분담

- **Fable (기획 총괄)**: 시나리오 스키마 확정, 챕터별 개요·핵심 분기·플래그 설계,
  외주 결과물 리뷰·통합·플레이스루 검증(분기 누락, 점수 밸런스, 톤 일관성).
- **Codex `gpt-5.6-terra` (병렬 외주, pumasi 방식)** — 검증 완료 (2026-07-16,
  `codex exec -m gpt-5.6-terra` 정상 동작 확인):
  1. engine.js + scoring.js 구현 (스키마 기반)
  2. 챕터 1~5 시나리오 데이터 집필 (챕터별 1인스턴스, 개요+스키마 지급)
  3. CSS/비주얼 (특강 덱 디자인 토큰 참조)
- 통합 후 Fable이 전 분기 자동 순회 검증 스크립트 + 수동 플레이스루로 검수.

## 7. 오류 처리·엣지 케이스

- 시나리오 데이터 로드 실패 시 명시적 오류 화면 (빈 화면 금지).
- 존재하지 않는 scene 참조는 엔진이 콘솔 경고 + 안전한 챕터 시작점으로 폴백.
- localStorage 불가 환경(시크릿 모드 등)에서는 이어하기 없이 정상 진행.
- 스키마 검증기(개발용)를 두어 하위모델 산출물의 구조 오류를 통합 전에 잡는다.

## 8. 검증 기준 (완료 정의)

1. 모든 scene 도달 가능, 모든 선택지의 다음 scene 존재 (자동 검증 스크립트).
2. 6D 각 축이 최소 4개 이상의 결정에서 변동 가능 (밸런스 검증).
3. 챕터 5의 레드팀 사건 6종이 각각 특정 플래그 조합으로 발화함을 확인.
4. 모바일(390px)·데스크톱에서 전체 플레이스루 정상.
5. GitHub Pages 배포 후 링크 접속만으로 플레이 가능.
