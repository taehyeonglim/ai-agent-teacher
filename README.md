당신은 한별초 5학년 2반 담임교사. 스스로 행동하는 AI 에이전트 "온새미"와 함께
3월 개학부터 여름방학까지 한 학기를 보내며, 매 순간 결정합니다 —
**무엇을 위임하고, 어디서 멈추게 하고, 누가 책임질 것인가.**

## 6D 모델

| 단계 | 정의 |
|---|---|
| **Define** 정의 | 교육목표와 인간이 책임질 영역을 정의한다 |
| **Design** 설계 | 인간과 AI의 전체 업무 흐름을 설계한다 |
| **Delegate** 위임 | AI에 맡길 업무·데이터·도구·권한을 정한다 |
| **Detect** 탐지 | 오류·편향·개인정보·저작권·과잉의존을 탐지한다 |
| **Decide** 결정 | 승인·수정·중단·재검토를 인간이 결정한다 |
| **Disclose** 공개 | AI 사용·출처·과정·책임자를 투명하게 밝힌다 |

> 6D 모델: UNESCO(2024)·EU AI Act 제4조·OECD를 종합한 임태형(전주교육대학교)의
> 재구성 실천 모델 (공인 프레임워크 아님)

## 특징

- 5챕터 + 에필로그, 결정 17~19회, 플레이 약 20분 (전 장면 삽화 67컷)
- 선택은 서사로만 돌아옵니다 — 게임 중 점수는 보이지 않습니다
- 앞 챕터의 선택이 마지막 챕터의 "사건"(레드팀 6종)으로 발화합니다
- 엔딩: 6D 레이더 + 플레이 스타일 칭호(6종 도감) + 30일 실천 처방 + 에필로그
- **복기하기**: 한 학기의 결정을 되짚고, 아쉬운 결정에선 그때의 다른 길을 다시 봅니다
- **결과 공유**: 결과 카드 PNG 저장, 결과 링크 복사(서버 없이 URL로 재현)
- 순수 정적 웹 (서버·API 불필요), 모바일 대응, `localStorage` 이어하기

## 개발

- 시나리오 데이터와 엔진 분리: `js/data/schema.md`가 계약서 — 새 시나리오 팩 제작은 [docs/scenario-pack-guide.md](docs/scenario-pack-guide.md)
- 구조 검증: `node tools/validate.js`
- 로직 테스트: `node --test tools/*.test.js` (CI가 push마다 자동 실행)
- 밸런스 점검: `node tools/playthrough.js` (3전략) · `node tools/montecarlo.js` (무작위 1,000회 분포)
- js/css 수정 배포 시 `index.html`의 `?v=N`을 1 올릴 것 (캐시버스팅)
